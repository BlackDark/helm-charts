CONFIGURATION REQUIRED:
======================

Two things need to be configured:

1. Static Configuration (config dict) - values written to ConfigMap
2. Secrets & External References (env array) - database/Redis connections

MINIMAL REQUIRED CONFIGURATION:
==============================

env:
  # Database - REQUIRED
  - name: DATABASE_HOST
    valueFrom:
      secretKeyRef:
        name: postgres-secret
        key: host
  - name: DATABASE_PASSWORD
    valueFrom:
      secretKeyRef:
        name: postgres-secret
        key: password
  
  # Redis - REQUIRED
  - name: REDIS_URL
    valueFrom:
      secretKeyRef:
        name: redis-secret
        key: url

# Optional: Auto-generate the SECRET_KEY_BASE secret
# secretKeyBase: "generate-a-secure-random-string-here"

# Optional: Customize application behavior
config:
  APPLICATION_HOSTS: "dawarich.example.com"
  TIME_ZONE: "America/New_York"
  DISTANCE_UNIT: "mi"

NEXT STEPS:
===========

1. Create Kubernetes secrets for your database and Redis:
   kubectl create secret generic postgres-secret \
     --from-literal=host=your-postgres-host \
     --from-literal=password=your-password \
     --from-literal=username=dawarich

   kubectl create secret generic redis-secret \
     --from-literal=url=redis://...

2. Optionally, generate the SECRET_KEY_BASE secret:
   kubectl create secret generic dawarich-app \
     --from-literal=secretKeyBase=$(openssl rand -hex 32)

   Or provide secretKeyBase in values.yaml to auto-generate it.

3. Update your values.yaml with the configuration above

4. Upgrade the Helm release:
   helm upgrade dawarich dawarich/dawarich -f values.yaml
